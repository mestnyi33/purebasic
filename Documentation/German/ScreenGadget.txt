@Library ScreenGadget

@Overview

  Die ScreenGadget-Bibliothek ermöglicht Ihnen die Erstellung
  komplexer grafischer Benutzeroberflächen (GUIs)
  direkt auf dem Bildschirm.
@LineBreak
  Die Syntax dieser Bibliothek ist ähnlich der
  in der @LibraryLink "gadget" "Gadget" Bibliothek.
@LineBreak
@LineBreak
  @@InitScreenGadgets muss aufgerufen werden,
  bevor die Befehle rund um die Gadgets verwendet werden können.

@CommandList

@ExampleFile All ScreenGadget.pb

@SupportedOS

;------------------------------------------------------------------------------------

@Function InitScreenGadgets()

@Description
  Initialisiert die ScreenGadget-Bibliothek.
 
@NoReturnValue
  
@Remarks  
  Muss nach @@OpenScreen / @@OpenWindowedScreen aufgerufen werden.

@SupportedOS

@Function RenderScreenGadgets()

@Description
  Zeigt Fenster und Gadgets auf dem Bildschirm an.
 
@NoReturnValue
  
@Remarks  
  Diese Funktion muss in die Rendering-Schleife eingefügt werden.
  Der Hintergrund muss "aufgefrischt" werden, zum Beispiel mit @ClearScreen oder @RenderWorld (3D).

@SupportedOS

;------------------------------------------------------------------------------------

@Function ScreenWindowAnimation(#ScreenWindow, Bedeutung, Dauer [, Fading [, Zoom, [,x [,y [, Rotation]]]])

@Description
  Lässt das Fenster mit einer Animation ein- bzw. ausblenden.
 
@Parameter "#ScreenWindow"
  Das zu animierende Fenster.

@Parameter "Bedeutung"
  1: Erscheinen
  2: Verschwinden
  3: Erscheinen und Verschwinden

@Parameter "Dauer"
  In Millisekunden.

@OptionalParameter "Fading"
  0 oder 1.

@OptionalParameter "Zoom"
  0 oder 1.

@OptionalParameter "x"
  -1: Das Fenster erscheint/verschwindet von links, 1: von links, 0: keine horizontale Bewegung.

@OptionalParameter "y"
  -1: Das Fenster erscheint/verschwindet von oben, 1: von unten, 0: keine vertikale Bewegung.

@OptionalParameter "Drehung"
  In Grad.

@NoReturnValue
 
@SupportedOS

@Function Ergebnis = ScreenVirtualSize(VirtuelleBreite, VirtuelleHöhe)

@Description
  Definiert die virtuellen Bildschirmabmessungen, um die Position von Gadgets automatisch an die Auflösung anzupassen.
  @LineBreak
  Dieser Befehl ist nützlich, wenn die Bildschirmauflösung nicht im Voraus bekannt ist.
	
@Parameter "VirtuelleBreite, VirtuelleHöhe"
  Die virtuellen Abmessungen des Bildschirms.
 
@ReturnValue
  Ein Fließkomma-Wert, entsprechend dem Minimum (tatsächliche Breite / virtuelle Breite, tatsächliche Höhe / virtuelle Höhe).
  Dieser Wert kann dann verwendet werden, um die Schriftgröße anzupassen.
  
@Remarks  
  Wenn dieser Befehl verwendet wird, müssen Position und Größe des Gadgets in diesen virtuellen Dimensionen definiert werden.

@Example
@Code
  ...
  fontratio.f = ScreenVirtualSize(1000, 500)
  SetScreenGadgetFont(#PB_Default, LoadFont(0, "Arial", 24*fontratio))
  ...
@EndCode

@SupportedOS

@Function Ergebnis = AddScreenGadgetItem(#ScreenGadget, Position, Text$)

@Description
  Fügt einen Eintrag zum angegebenen #ScreenGadget hinzu.
  
@Parameter "#ScreenGadget"
  Das zu verwendende Gadget.
  
@Parameter "Position"
  Der Eintrag-Index, wo der neue Eintrag eingefügt werden soll. Um diesen Eintrag am Anfang einzufügen,
  benutzen Sie einen Index von 0. Um diesen Eintrag am Ende der aktuellen Eintrag-Liste hinzuzufügen,
  benutzen Sie einen Wert von -1. Denken Sie beim Einfügen eines Eintrags daran, dass alle aktuellen
  Einträge, die auf diesen neuen Eintrag folgen, einen um 1 höheren Positions-Index erhalten.
  
@Parameter "Text$"
  Der Text für den neuen Eintrag.
  
@NoReturnValue
  
@Remarks
  Die folgenden Gadgets werden unterstützt:
  @LineBreak
  @LineBreak
  - @@ComboBoxScreenGadget @LineBreak
  - @@ListViewScreenGadget @LineBreak
  - @@PanelScreenGadget @LineBreak
  @LineBreak

@SeeAlso
  @@ComboBoxScreenGadget, @@ListViewScreenGadget, @@PanelScreenGadget

@SupportedOS

@Function Ergebnis = ContainerScreenGadget(#ScreenGadget, x, y, Breite, Höhe)

@Description
  Erstellt ein Container Gadget innerhalb der aktuellen Gadgetliste.
@LineBreak
@LineBreak
  Es ist ein simples Panel-Gadget
  (Schalttafel), welches andere Gadgets enthalten kann. Sobald dieses Gadget erstellt wurde,
  werden alle nachfolgenden Gadgets innerhalb des Containers erstellt. Wenn alle benötigten
  Gadgets erstellt wurden, muss @@CloseScreenGadgetList aufgerufen werden, um zur vorherigen
  Gadgetliste zurückzukehren. @@OpenScreenGadgetList kann später benutzt werden, um weitere
  Gadgets während des Programmablaufs ("on-the-fly") zum Container-Bereich hinzuzufügen.

@Parameter "#ScreenGadget"
  Eine Nummer zum Identifizieren des neuen Gadgets.
  @ReferenceLink "purebasic_objects" "#PB_Any" kann verwendet werden, um diese Zahl
  automatisch zu generieren.

@Parameter "x, y, Breite, Höhe"
  Die Position und Abmessungen des neuen Gadgets.

@ReturnValue
  Gibt im Erfolgsfall einen Wert ungleich Null zurück, andernfalls Null. Wenn @#PB_Any
  als #ScreenGadget Parameter verwendet wurde, dann enthält bei Erfolg der Rückgabewert
  die automatisch generierte Nummer.

@Remarks
  Zu diesem Gadget kann mittels @@ScreenGadgetToolTip eine 'Mini-Hilfe' hinzugefügt werden.

@SupportedOS

@LineBreak
@LineBreak
  Es ist sehr nützlich, wenn ein Gadget zu groß ist, um in die Fenster-Dimensionen zu passen.
  In diesem Fall kann es in den ScrollArea-Bereich gelegt werden. Das gesamte Scrolling wird automatisch
  vom Gadget verwaltet. Dies ist ein Container-Gadget und dafür gedacht, ein oder mehrere 
  Gadgets innerhalb seines Scroll-Bereiches zu haben. Sobald dieses Gadget erstellt wurde,

  zur vorherigen Gadgetliste zurückzukehren. @@OpenScreenGadgetList kann später verwendet
  werden, um weitere Gadgets während des Programmablaufs ("on-the-fly") zum Scroll-Bereich
  hinzuzufügen.

@LineBreak 
  ScreenWindowEvent() gibt das nächste Ereignis aus dem "Ereignis-Queue" (Warteschlange mit Ereignissen)
  zurück bzw. Null, wenn es keine weiteren Ereignisse gibt.
  Die Funktion wartet nicht auf das nächste Ereignis - sie kehrt immer sofort zurück.
  
@NoParameters

@ReturnValue
  Mögliche Ereignisse sind:
@FixedFont
  @#PB_Event_Gadget         : ein @LibraryLink "gadget3d" "ScreenGadget" wurde gedrückt
  @#PB_Event_CloseWindow    : das Schließgadget vom Fenster wurde gedrückt
  @#PB_Event_SizeWindow     : das Fenster wurde in der Größe verändert
  @#PB_Event_MoveWindow     : das Fenster wurde verschoben
  @#PB_Event_ActivateWindow : das Fenster wurde aktiviert (hat den Fokus erhalten)
@EndFixedFont

@SeeAlso
  @@InputEvent
